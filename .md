## 0. Project Vision

A web application where any user can upload a CSV or Excel file and immediately get:
- Auto-generated KPI cards, trend charts, and bar charts
- AI-powered natural language Q&A ("Why did revenue drop in March?")
- Anomaly detection that flags outliers automatically
- A downloadable PDF report with an AI-written executive summary

**The bar**: It should look and feel like a premium Apple product — smooth, intentional, and beautiful. Every interaction must feel considered. No clutter, no noise, no generic SaaS grey.

## 3. UI / UX Design System — Apple-Grade Rules

> These rules apply to every single component. No exceptions.

### 3.1 Typography
- Font: `Inter` (variable, loaded via `next/font/google`)
- Scale: 12 / 14 / 16 / 20 / 24 / 32 / 48px only
- Weights: 400 (body), 500 (labels), 600 (headings), never 700+
- Letter spacing: `-0.02em` on all headings 20px and above
- Line height: `1.5` body, `1.2` headings

### 3.3 Component Aesthetic Rules
- Every card: `background: var(--bg-surface)`, `border: 1px solid var(--border)`, `border-radius: var(--radius-lg)`, NO box-shadow on base cards
- Hover state on interactive cards: `transform: translateY(-2px)`, `box-shadow: 0 8px 32px rgba(0,0,0,0.08)`, transition `200ms ease`
- Active/pressed state: `transform: scale(0.98)`, `200ms ease`
- All transitions: `transition: all 200ms cubic-bezier(0.25, 0.46, 0.45, 0.94)`
- Loading skeletons: `animate-pulse` with `bg-[var(--bg-base)]`
- Focus rings: `outline: 2px solid var(--accent-blue)`, `outline-offset: 2px`

### 3.4 Motion Principles
- Page load: elements stagger in with `opacity: 0 → 1` + `translateY(16px → 0)`, 60ms stagger between each card
- Number counters: animate from 0 to final value over 800ms with `easeOut`
- Charts: animate bars/lines in on mount (Recharts has this built in — always enable it)
- AI response: stream text character by character (do not dump the whole response at once)
- Tab/view switches: cross-fade, `200ms`
- Modals: scale in from `0.95 → 1` + `opacity 0 → 1`, `250ms`

## 4. Backend Build Plan (FastAPI)

### Step B1 — Project Initialization

Create `main.py`:
```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from routers import upload, analyse, query

app = FastAPI(title="AI BI Dashboard API")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000", "https://your-vercel-app.vercel.app"],
    allow_methods=["*"],
    allow_headers=["*"],
)

app.include_router(upload.router, prefix="/api")
app.include_router(analyse.router, prefix="/api")
app.include_router(query.router, prefix="/api")
```

### Step B2 — File Upload Router (`routers/upload.py`)
- Accept `multipart/form-data` with a single file field named `file`
- Validate: only `.csv` and `.xlsx` allowed, max 50MB
- Parse with `services/parser.py` → get column names, dtypes, row count, sample rows
- Store raw file in Supabase Storage bucket `datasets`
- Store metadata (filename, schema, row_count, upload_time) in Supabase table `datasets`
- Return `{ id, filename, schema, row_count, preview_rows }`

### Step B3 — Parser Service (`services/parser.py`)
```python
# Rules:
# 1. Auto-detect delimiter for CSVs (comma, semicolon, tab)
# 2. Auto-detect date columns (try parsing, if >80% parse → mark as date)
# 3. Auto-detect numeric columns (strip currency symbols, commas)
# 4. Return: { columns: [{name, dtype, sample_values}], row_count, preview: first 10 rows }
```

### Step B4 — Analyser Service (`services/analyser.py`)
Compute all of the following automatically from the DataFrame:

**KPIs (auto-select based on dtypes):**
- If numeric columns exist: sum, mean, min, max for top 3 numeric columns
- If date column exists: date range, records per time period
- Always compute: row count, column count, null percentage

**Charts:**
- Time series: if date + numeric → generate line chart data (group by month/week)
- Distribution: top 5 values of the first categorical column → bar chart
- Correlation: if 2+ numeric columns → scatter data

**Anomaly Detection:**
- For each numeric column: compute IQR, flag values outside `Q1 - 1.5*IQR` and `Q3 + 1.5*IQR`
- Return list of `{ column, value, row_index, severity: 'high'|'medium' }`

### Step B5 — AI Query Service (`services/ai_service.py`)
```python
import anthropic

client = anthropic.Anthropic()

def build_system_prompt(schema, sample_data, kpis):
    return f"""
You are a senior business analyst AI embedded in a BI dashboard.
You have access to a dataset with the following schema:
{schema}

Key metrics already computed:
{kpis}

Sample data (first 20 rows):
{sample_data}

Rules:
- Answer questions about this dataset directly and concisely
- Cite specific numbers from the data
- If you spot a trend or anomaly, mention it proactively
- Keep answers under 150 words unless the user explicitly asks for more
- Format numbers nicely (€1.2M not 1200000)
- Never make up data that isn't in the dataset
"""

def stream_query(question: str, schema: dict, sample_data: str, kpis: dict):
    with client.messages.stream(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        system=build_system_prompt(schema, sample_data, kpis),
        messages=[{"role": "user", "content": question}]
    ) as stream:
        for text in stream.text_stream:
            yield text
```

### Step B6 — Query Router (`routers/query.py`)
- POST `/api/query`
- Body: `{ dataset_id: string, question: string }`
- Fetch dataset from Supabase, load into Pandas
- Call `ai_service.stream_query()`
- Return as `StreamingResponse` with `media_type="text/event-stream"`

```typescript
// Use this hook in KPICard.tsx
function useCountUp(target: number, duration: number = 800) {
  const [value, setValue] = useState(0);
  useEffect(() => {
    const start = performance.now();
    const tick = (now: number) => {
      const progress = Math.min((now - start) / duration, 1);
      const eased = 1 - Math.pow(1 - progress, 3); // easeOutCubic
      setValue(Math.round(target * eased));
      if (progress < 1) requestAnimationFrame(tick);
    };
    requestAnimationFrame(tick);
  }, [target, duration]);
  return value;
}
```

**Query flow:**
1. User submits question
2. Input disables, shows subtle spinner inside send button

4. AI response streams in — append text token by token (use `ReadableStream` from SSE response)

6. Each response stored in a local history array — scrollable carousel above the input

**Streaming response implementation in `useAIQuery.ts`:**
```typescript
async function queryAI(datasetId: string, question: string, onChunk: (text: string) => void) {
  const response = await fetch('/api/query', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ dataset_id: datasetId, question }),
  });
  const reader = response.body?.getReader();
  const decoder = new TextDecoder();
  while (reader) {
    const { done, value } = await reader.read();
    if (done) break;
    onChunk(decoder.decode(value));
  }
}
```

## 6. Database Schema (Supabase)

### Table: `datasets`
```sql
create table datasets (
  id          uuid primary key default gen_random_uuid(),
  user_id     uuid references auth.users,
  filename    text not null,
  file_path   text not null,              -- Supabase Storage path
  row_count   integer,
  schema      jsonb,                       -- [{name, dtype, sample_values}]
  kpis        jsonb,                       -- computed KPIs
  anomalies   jsonb,                       -- detected anomalies
  created_at  timestamptz default now()
);
```

### Table: `queries`
```sql
create table queries (
  id          uuid primary key default gen_random_uuid(),
  dataset_id  uuid references datasets(id),
  question    text not null,
  answer      text not null,
  created_at  timestamptz default now()
);
```

Enable Row Level Security (RLS) on both tables so users only see their own data.
